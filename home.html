Responsive Redesign

Overview

This project practices the workflow of redesigning a website. The pipeline is dedicated to analyzing and identifying flaws in an existing interface, creating low-fidelity and high-fidelity prototypes for various screen sizes, and building a responsive website based on those prototypes.

Usability Problems

The original website is the online ordering page of a local restaurant in Providence. I analyzed the usability of the website using the following criteria: learnability, memorability and efficiency.

I further analyzed the accessibility using WebAIM WAVE. Most of the alerts and errors lie in the very low contrast between text and background colors, and the small text which is hard to read. Some other structural warnings such as skipped heading level and missing alternative text also exist.

Visual Redesign

Based on the problems identified above, I created low-fidelity wireframes, high-fidelity prototypes and a visual design style guide. The screen is designed to be responsive with different layouts for mobile, tablet and desktop. Low-fi wireframes are annotated how they address the usability problems described earlier and high-fi prototypes are annotated how they satisfy the needs for responsive design and interactions with other pages. In this session, I also create a visual design style guide documenting interactive elements to keep their interactions consistent throughout the webpage.

Responsive Redesign

Following the steps above, I finally created the webpage using HTML and CSS. The webpage is designed to be responsive for the range of view on laptop, tablet and mobile. For example, on tablet and mobile devices, there are no margins on the side, and the layout of main menu columns differ. The collapsed navbar will be in the format of hamburger menu on mobile. The font size will also change responsively on different devices.
